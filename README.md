# Fasting Companion Mini Project 1 

Fasting Companion is a mini project built to support my friend Kike during fasting periods.

The goal is to make fasting guidance simple, visual, and practical: choose a fasting duration, follow hydration recommendations, review safety reminders, and track the fast with a live countdown.

## Project Purpose

This project focuses on solving a real user need:
- safer fasting planning,
- better hydration awareness,
- clear step-by-step flow,
- simple mobile-friendly experience.

## Features

- Multi-step fasting setup flow
- Duration options: 1, 2, 3, or 5 days
- Hydration tracker with selectable intake levels
- Safety confirmation before starting a fast
- Full-screen active fast page (pop-out style)
- Live countdown timer based on selected duration
- Progress and water goal indicators
- Post-fast nutrition reminder message

## Tech Stack

- HTML
- CSS
- JavaScript (for countdown and dynamic values)

## Notes

- All interface images are stored locally in [images](images).
- The color theme was updated to a more professional yellow/green style.
- The app is designed as a learning mini project, not a medical device.

## What I Learned

- Interviewing first changes the quality of the final product because the flow is built around a real person’s needs, not assumptions.
- CSS-only step logic works well for structure, but JavaScript is still necessary for real-time features like countdown timers and synced progress values.
- Professional visual design is mostly about consistency: spacing, typography, color harmony, and readable contrast.
- AI helps speed up implementation, but the best results came from reviewing, testing, and rewriting suggestions to match my client’s goals.
